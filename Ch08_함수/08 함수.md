### 08 함수

#### 프로그래밍에서 함수의 사용 목적

**1. 코드 재사용 (Reusability)**
같은 작업을 여러 번 수행해야 할 때, 함수로 만들어두면 반복해서 코드를 작성할 필요 없이 필요할 때마다 호출만 하면 됩니다.

**2. 코드 구조화와 가독성 향상**
복잡한 프로그램을 작은 단위(함수)로 나누면 코드가 논리적으로 정리되어 이해하기 쉬워집니다. 각 함수가 하나의 명확한 역할을 담당하게 됩니다.

**3. 유지보수 용이성**
로직이 한 곳(함수 내부)에 모여 있으므로, 수정이 필요할 때 그 함수만 고치면 프로그램 전체에 반영됩니다. 반대로 함수 없이 코드를 여러 곳에 복사-붙여넣기 했다면, 모든 곳을 일일이 찾아 수정해야 합니다.

**4. 추상화 (Abstraction)**
함수를 사용하는 쪽에서는 내부 구현을 몰라도 됩니다. 함수의 이름과 입력/출력만 알면 사용할 수 있어, 복잡한 세부 사항을 숨기고 "무엇을 하는지"에만 집중할 수 있습니다.

**5. 테스트와 디버깅 용이성**
기능이 작은 단위로 분리되어 있으면 각 함수를 독립적으로 테스트하기 쉽고, 오류가 발생했을 때 문제의 범위를 좁혀 원인을 찾기도 수월합니다.

**6. 네임스페이스 분리**
함수 내부에서 선언한 변수는 기본적으로 그 함수 안에서만 유효(지역 변수)하므로, 프로그램 전체의 변수 이름이 충돌하는 것을 방지할 수 있습니다.
