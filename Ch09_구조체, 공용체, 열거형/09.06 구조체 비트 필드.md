### 09.06 구조체 비트 필드

#### 구조체 비트필드(bit field)란?

비트필드는 쓸데없이 많은 메모리를 차지하는 데이터를 줄여서 꼭 필요한 공간만 할당하여 메모리 공간을 절약하고자 할 때 주로 사용한다.

또는 한 비트씩 필요한 데이터를 사용하고자 할 때 사용할 수도 있다.

CPU내의 상태 레지스터와 유사한 방법으로 사용할 수도 있다.

컴퓨팅 환경이 열악한 곳, 즉 메모리 공간이 많지 않은 임베디드 시스템 같은 곳에서 사용하면 유용하다.

#### 구조체 비트필드 선언방법
```C
struct 구조체명 {
	멤버데이터형 구조체멤버1:비트수
	멤버데이터형 구조체멤버2:비트수
	멤버데이터형 구조체멤버3:비트수
	...
};
```

변수 정의는 구조체와 동일하고, 엑세스 방법도 구조체와 동일하다.

---
#### ● 예제 09-06-1 : 구조체 비트 필드를 이용한 날짜 시간 표현
```C
1 #include <stdio.h>
2 
3 typedef struct {
4 	unsigned int year:12;	// 12bit: 0~4095
5 	unsigned int month:4;	// 4bit: 0~15
6 	unsigned int day:5;	// 5bit: 0~31
7 	unsigned int hour:5;	// 5bit: 0~31
8 	unsigned int min:6;	// 6bit: 0~63
9 } schedule;
10 
11 int main()
12 {
13 	schedule my;		// 구조체 변수 선언
14 	my.year = 2001;		// 비트필드에 값 저장
15 	my.month = 5;
16 	my.day = 28;
17 	my.hour = 12;
18 	my.min = 30;
19 	printf("%d년 %d월 %d일 %d시 %d분\n",
20 		my.year, my.month, my.day, my.hour, my.min); 
21 	printf("구조체의 크기 : %dByte\n", sizeof(schedule));
22 }
```

#### ● 해설
* 4\~10: 구조체 비트 필트 선언. 각 필드를 해당하는 비트만큼만 지정한다. 이렇게 함으로써 데이터가 저장될 공간을 절약할 수 있다. 이 정도면 충분히 공간으로 사용할 수 있다. 만약 이들을 각각 int로 선언한다면 전체 공간은 20바이트가 필요하지만 여기서는 4바이트이면 충분하다.
* 14\~18: 각 필드에 값을 저장한다.
* 19\~20: 각 필드에 저장된 값을 출력한다.
* 21: 이 구조체의 크기를 출력한다. 4바이트인 것을 확인할 수 있다.
> [!NOTE]
> 비트필드의 사용은 구조체와 동일하다. 단 비트필드 구조체를 정의할 때는 그 크기를 잘 계산하여 낭비되지 않도록 한다. 그리고 자료형도 적당히 설정한다. 여기서는 unsigned int로 하였다. 즉, 음수 값이 없으므로 모두 unsigned로 한 것이다. unsigned로 하지 않으면 2의 보수 표현 범위내에서만 데이터 값이 결정된다. 2의 보수의 표현 범위:, n은 비트 수이다.

#### ● 예제 09-06-2 : 구조체 비트 필드를 이용한 학년, 반, 번호 표시
```C
1 #include <stdio.h>
2 
3 typedef struct {
4 	unsigned short Grade:4;		// 학년 4비트
5 	unsigned short Class:5;		// 반 5비트
6 	unsigned short Number:7;		// 번호 7비트
7 } student;
8 
9 int main()
10 {
11 	unsigned char x;
12 	student s;
13 	printf("학년 입력(1~12): ");
14 	scanf("%u", &x);		// 비트 필드는 개별적으로 주소가 없으므로
15 	s.Grade = x;		// 다른 변수에 읽어서 다시 비트필드에 저장
16 				// 즉, 비트필드는 직접 scanf에 사용할 수 없음
17 	printf("반 입력(1~31): ");
18 	scanf("%u", &x); 
19 	s.Class = x;
20 	printf("번호 입력(1~127): ");
21 	scanf("%u", &x);
22 	s.Number = x;
23 	printf("%u학년 %u반 %d번\n", s.Grade, s.Class, s.Number);
24 	s.Grade = 10, s.Class = 15, s.Number = 20;
25 	printf("%u학년 %u반 %d번\n", s.Grade, s.Class, s.Number);
26 	printf("구조체의 크기: %dByte\n", sizeof(student));
27 }
```

#### ● 해설
* 3\~7: 구조체 비트 필트 선언. 각 필드를 해당하는 비트만큼만 지정한다. unsigned short는 16비트이다. 이들 16비트를 3개의 비트필드로 나누어 저장한다.
* 12: 비트 필드 변수명 선언
* 14: 데이터를 읽는다.
* 15: 읽은 데이터를 비트 필드 멤버에 저장한다.
* 23: 저장된 값을 출력한다.
* 24\~25: 다른 데이터를 저장하고, 출력한다.
* 26: 비트필드가 차지하는 공간의 크기가 얼마인지 출력한다.
> [!NOTE]
> 비트 필드가 차지하는 공간보다 큰 데이터가 입력되면 앞부분이 잘려버린다. 즉, 2진수로 나타낼 때 뒷부분만 해당 비트만큼 저장된다.

#### ● 예제 09-06-3 : 구조체 비트 필드를 이용한 학사정보 저장
```C
1 #include <stdio.h>
2 #include <stdlib.h>
3 
4 struct student {
5 	char name[30];
6 	unsigned char college:4;	// 단과대학(학부) : 4비트 : 16종류 
7 	unsigned char major:4;	// 학과(전공) : 4비트 : 16종류 
8 	unsigned char grade:3;	// 학년 : 3비트

10 		// 편입/전과 : 2비트 : 0:해당없음, 1:전과, 2:편입, 3:재입학

13 	unsigned char credits:8; 	// 이수학점 : 8비트 
14 	unsigned char gender:1;	// 성별 : 1비트 ; 0:남자, 1:여자 
15 	unsigned char military:2;	// 군 : 2비트 : 0:면제, 1:미필, 2:군필

18 	       // 충북:9, 충남:10, 전북:11, 전남:12, 경북:13, 경남:14, 제주:15, 기타:16 
19 };
20 
21 typedef struct student student;

24 enum Collages {Law, Human, SSci, Busi, NSci, Nurs, Eng, Arg, Art, Edu,
25 		Home, Vet, Phar, Mus, Med };
26 char *Gender[] = {"남자", "여자"}; 
27 enum Genders {Male, Female };

29 enum States {ADM, GA, MA, RI, GR, COMP, LEV, REM};
30 char *Mil[] = {"면제", "미필", "군필"}; 
31 enum Military {Na, Incomp, Veteran};
32 char *Trans[] = {"해당없음", "전과", "편입", "재입학"}; 
33 enum Transfers { NA, CM, SE, RE };
34 
35 char *Region[] = {"서울특별시", "부산광역시", "대구광역시", "인천광역시", "광주광역시",
36 	    "대전광역시", "울산광역시", "경기도", "강원도",  "충청북도", "충청남도",
37 	    "전라북도", "전라남도", "경상북도", "경상남도", "제주특별자치도", "해외"};

48 	printf("%s : ", s.name);
49 	printf("%s대학, 전공:%u, %u학년, %s, %s\n",
```

#### ● 해설
* 5\~20: 구조체 비트 필트 선언. 각 필드를 해당하는 비트만큼만 지정한다. 10개의 필드로 구성한다. 이름을 제외한 모든 데이터들은 비트별로 나누어 저장한다.
* 22: 새로운 데이터 형을 지정한다.
* 23\~26: 단과대학별로 해당하는 이름과 열거형을 지정한다.
* 27\~28: 성별에 대한 이름과 열거형을 지정한다.
* 29\~30: 현재 학적 상태에 대한 이름과 열거형을 지정한다.
* 31\~32: 병역 상태에 대한 이름과 열거형을 지정한다.
* 33\~34: 학전 변동에 대한 이름과 열거형을 지정한다.
* 36\~41: 주소지역에 대한 이름과 열거형을 지정한다.
* 42: 학년에 대한 열거형을 지정한다.
* 45: 비트 필드의 변수를 선언하고 초깃값을 지정한다.
* 47: 구조체의 크기를 출력해 본다. 이름을 제외하고 나면 4바이트이면 충분히 데이터를 저장할 수 있음을 알 수 있다.
* 49\~52: 저장된 값을 출력한다. 해당되는 문자열을 출력한다.
> [!NOTE]
> enum으로 선언된 모든 기호들은 같은 것이 있으면 안 된다.
