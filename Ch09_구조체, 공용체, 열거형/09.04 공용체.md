#### 09.04 공용체

공용체(union)이란 동일한 공간에 서로 다른 형태의 데이터를 저장한다. 따라서 한 시점에는 하나의 형태로만 저장된다.
* int 형태로 저장된 변수를 double 형태로 읽으면 원하지 않는 결과가 출력된다.
* 이를 역이용하여 프로그램을 작성할 수도 있다.

#### 공용체 선언 방법
```C
enum enum_name { 멤버1, 멤버2, ... };
변수 선언 : enum enum_name enum_var
```

---
#### ● 예제 09-04-1 : 공용체 예, 정수와 실수를 같은 장소에 저장하기
```C
1 #include <stdio.h>
2 
3 typedef union int_or_float { 
4 	int i; 
5 	float f; 
6 } number;
7 
8 int main()
9 {
10 	number n; 
11 	n.i = 1234; 	// 정수 값 대입 
12 	printf("i:%10d,  f:%16.10e\n", n.i, n.f);	// 정수 형만 정확한 값을 보여준다. 
13 	n.f = 1234.0f; 	// 실수 값 대입 
14 	printf("i:%10d,  f:%16.10e\n", n.i, n.f);	// 실수 형만 정확한 값을 보여준다. 
15 }
```

#### ● 해설
3~6: 두 개의 멤버를 가지는 공용체를 정의하고, 이 공용체를 다시 새로운 데이터 형 number로 선언한다. 두 멤버가 선언되었지만 실제로는 한 공간을 두 멤버가 공용한다. 즉, 한 시점에서는 한 멤버만 사용하게 된다.
10: 공용체 number로 변수 n을 선언
11: 공용체 n의 멤버 i에 정수값 1234 대입. 정수 값 멤버에 값이 대입되면 정수형으로 사용된다.
12: 따라서 정수값만 정상적으로 출력되고 실수는 알 수 없는 값이 출려됨을 알 수 있다. 정수와 실수는 메모리에 저장되는 방식이 다르므로 메모리에서 읽어내는 방법도 다르다. 따라서 정상적으로 저장된 값만이 정상적으로 출력된다.
13: 공용체 n의 멤버 f에 실수 값 1234.0f가 대입. 실수 값 멤버에 값이 대입되면 실수형으로 사용된다. 1234.0f에서 f는 float값이라는 것을 의미한다. f가 없으면 실수의 기본형인 double로 인식한다.
14: float값만 정상적으로 출력된다.


#### ● 예제 09-04-2 : 실수를 2진수로 변환하여 보여주기
1 #include <stdio.h> 
2 #include <stdlib.h>
3 
4 typedef union int_or_float { 
5 	int i;
6 	float f;
7 } number;
8 
9 int main()
10 {
11 	number n;
12 	static char num[33];
13 
14 	n.i = 236523; 		// 정수 대입 
15 	itoa(n.i, num, 2);    // itoa는 표준함수는 아니지만 거의 대부분 지원
16 	printf("%10d -> %032s\n", n.i, num);
17 	// 32자리로 출력하고 앞의 빈자리에 0을 출력한다. 
18 	n.f = 20.5f; 		// 실수 대입 
19 	itoa(n.i, num, 2);		// 실수가 아닌 정수를 2진수로 변환 
20 	printf("%10.4f -> %032s\n", n.f, num);
21 
22 	n.f = -0.2f; 		// 실수 대입 
23 	itoa(n.i, num, 2);		// 실수가 아닌 정수를 2진수로 변환 
24 	printf("%10.4f -> %032s\n", n.f, num);
25 } // 원칙적으로 실수는 직접 2진수로 나타내기가 힘들지만 이와 같이 가능하다.

#### ● 해설
4~7: 두 개의 멤버를 가지는 공용체를 정의하고, 이 공용체를 다시 새로운 데이터 형 number로 선언한다.
12: 2진수가 문자열로 저장될 문자열 변수
14: 정수를 공용체에 대입
15,16: itoa를 사용하여 정수를 2진수로 변환하여 출력한다.
18: 실수를 공용체에 대입
19,20: itoa를 사용하여 정수를 2진수로 변환하여 출력한다. 실수가 아닌 정수를 2진수로 변환한다. 즉, 메모리 공간은 동일하므로 정수로 취급하여 2진수로 변환한다. itoa는 실수를 취급하지 않으므로 정수를 넘긴다. 그 결과는 메모리에 저장된 실수를 그대로 정수인 것처럼 취급하여 2진수로 변환하여 보여준다.
22,23: 음수인 실수값을 공용체에 대입하여 결과를 2진수로 변환하여 출력한다.
Notice: 실수(float)가 메모리에 저장되는 순서는 부호 1비트, 지수 8비트, 가수 23비트이다. 부호는 0이면 양수, 1이면 음수, 지수는 bias 127을 더하여 표시한다. 가수는 1.××××로 표시하며, “1.”은 생략한다. 예를 들어 실행 결과의 20.5를 실수로 나타내면, 부호는 0, 지수는 127이 더해져 있으므로 10000011(131)에서 01111111(127)을 빼면 4이다. 가수는 1.01001이다. “1.”은 생략되어 있으므로 “1.”을 넣어서 표시한다. 


#### ● 예제 09-04-3 : 세 가지 종류의 물건을 하나의 공용체로 표현하기
1 #include <stdio.h> 
2 #include <stdlib.h>
3 #include <string.h>
4 
5 struct Flower {	// 꽃의 속성 설정
6 	char name[30];
7 	enum {red, white, yellow, pink, etc} color;
8 };
9 
10 struct Fruit {		// 과일의 속성 설정
11 	char name[30];
12 	int calories;
13 };
14 
15 struct Vegetable {	// 채소의 속성 설정
16 	char name[30];
17 	int calories;
18 	int cooking_time;
19 };  
20 
21 union flower_fruit_vegetable {	//꽃, 과일, 채소를 하나의 공용체로 선언
22 	struct Flower flower;
23 	struct Fruit fruit;
24 	struct Vegetable vegetable;
25 }; 
26 
27 typedef struct {	// 물건의 종류와 공용체를 저장할 수 있는 구조체로 선언
28 	char type;
29 	union flower_fruit_vegetable ffv;
30 } things;
31 
32 enum type {flower, fruit, vegetable};
33 char *color[] = {"red", "white", "yellow", "pink", "etc"};
34 
35 int main ()
36 {
37 	int i;
38 	things x[3];
39 	x[0].type = flower;
40 	strcpy(x[0].ffv.flower.name, "Sunflower");
41 	x[0].ffv.flower.color = yellow;
42 	x[1].type = fruit;
43 	strcpy(x[1].ffv.fruit.name, "Apple");
44 	x[1].ffv.fruit.calories = 52;
45 	x[2].type = vegetable;
46 	strcpy(x[2].ffv.vegetable.name, "Carrot");
47 	x[2].ffv.vegetable.calories = 41;
48 	x[2].ffv.vegetable.cooking_time = 5;
49 	for(i=0; i<3; i++) {
50 		if(x[i].type == flower)	// type이 꽃이면 꽃에 맞는 내용 출력
51 			printf("%s, %s\n", x[i].ffv.flower.name,
52 				color[x[i].ffv.flower.color]);
53 		else if(x[i].type == fruit) // type이 과일이면 꽃에 맞는 내용 출력
54 			printf("%s, %dKcal\n", x[i].ffv.fruit.name,
55 				x[i].ffv.fruit.calories);
56 		else if(x[i].type == vegetable)	// type이 채소면 꽃에 맞는 내용 출력
57 			printf("%s, %dKcal, %dMin.\n",
58 				x[i].ffv.vegetable.name,
59 				x[i].ffv.vegetable.calories,
60 				x[i].ffv.vegetable.cooking_time);
61 	}
62 }

#### ● 해설
5~8: 꽃에 대한 구조체 선언
10~13: 과일에 대한 구조체 선언
15~19: 채소에 대한 구조체 선언
21~25: 이들 세 가지 구조체를 모두 공용으로 사용하기 위한 공용체 선언
27~30: 공용체의 구분을 위한 type 및 공용체를 멤버로 하는 새로운 구조체 선언
32: 세 가지 type에 대한 열거형 선언
33: 색상을 나타내기 위한 문자열 배열 선언
38: 구조체 배열 선언
39~48: 각 구조체의 값 지정
49~61: type에 따른 배열에 저장된 데이터 출력
