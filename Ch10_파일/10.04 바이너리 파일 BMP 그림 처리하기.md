### 10.04 바이너리 파일 BMP 그림 처리하기

#### ● 예제 10-04-1 : 그림파일인 BMP를 읽어 원래 그림의 역상을 파일로 저장하기
```C
1 #include <stdio.h>
2 #include <stdlib.h>
3 
4 int main()
5 {
6 	int i, n, adr;
7 	unsigned char tt[60], buf[16], file[20];
8 	FILE *fpi, *fpo;
9 
10 	printf("입력할 파일이름 : ");
11 	scanf("%s", file);
12 
13 	if((fpi = fopen(file, "rb")) == NULL) {
14 		printf("File not found\n", file);
15 		exit(0);
16 	}
17 
18 	if((fpo = fopen("reverse.bmp", "wb")) == NULL) {
19 		printf("File cannot create\n");
20 		exit(0);
21 	}
22 
23 	fread(tt, 54, 1, fpi);
24 	fwrite(tt, 54, 1, fpo);
25 
26 	while( fread(buf, 16, 1, fpi) != 0 ) {
27 		for(i=0; i < 16; i++)
28 			buf[i] = ~buf[i];
29 		fwrite(buf, 16, 1, fpo);
30 	}
31 	printf("File created!\n");
32 	fclose(fpi);
33 	fclose(fpo);
34 }
```

#### ● 해설
* 8: 파일 포인터 fpi, fpo 선언
* 11: 읽을 파일 이름을 키보드에서 받는다.
* 13\~16: 파일을 2진 읽기 모드로 열고, 파일이 존재하지 않으면 에러 메시지를 출력한다.
* 18\~21: 파일을 2진 쓰기 모드로 열고, 파일을 만들 수 없으면 에러 메시지를 출력한다.
* 23: 파일에서 BMP의 헤더 54바이트를 읽는다.
* 24: 파일에 BMP의 헤더 54바이트를 쓴다.
* 26\~30: 16바이트씩 읽은 다음 한 바이트씩 역상을 만들어 다른 파일에 저장한다.
* 26: 16바이트를 NULL이 아닌 동안 읽는다.
* 28: 한 바이트씩 역상을 만든다.
* 29: 16바이트를 출력 파일에 저장한다.
* 32\~33: 읽기, 쓰기 파일을 닫는다.

#### ● 예제 10-04-2 : BMP파일 두 개를 비트 &연산을 하여 새 파일로 저장하기
```C
1 #include <stdio.h>
2 #include <stdlib.h>
3 
4 int main()
5 {
6 	int i, n, adr;
7 	unsigned char tt[60], buf[32], buf2[32], file[20];
8 	FILE *fp1, *fp2, *fpo;
9 
10 	if((fp1 = fopen("t1.bmp", "rb")) == NULL) {
11 		printf("File not found\n", file);
12 		exit(0);
13 	}
14 
15 	if((fp2 = fopen("t2.bmp", "rb")) == NULL) {
16 		printf("File not found\n", file);
17 		exit(0);
18 	}
19 
20 	if((fpo = fopen("t1_AND_t2.bmp", "wb")) == NULL) {
21 		printf("File cannot create\n");
22 		exit(0);
23 	}
24 
25 	fread(tt, 54, 1, fp1);
26 	fread(tt, 54, 1, fp2);
27 	fwrite(tt, 54, 1, fpo);
28 	while( fread(buf,32,1,fp1) != 0 && fread(buf2,32,1,fp2) != 0) {
29 		for(i=0; i < 32; i++)
30 			buf[i] &= buf2[i];
31 		fwrite(buf, 32, 1, fpo);
32 	}
33 	printf("File created!\n");
34 	fclose(fp1);
35 	fclose(fp2);
36 	fclose(fpo);
37 }
```

#### ● 해설
* 8: 파일 포인터 fp1, fp2, fpo 선언
* 10\~13: “t1.bmp” 파일을 2진 읽기 모드로 열고, 파일이 존재하지 않으면 에러 메시지를 출력한다.
* 15\~18: “t2.bmp” 파일을 2진 읽기 모드로 열고, 파일이 존재하지 않으면 에러 메시지를 출력한다.
* 20\~23: “t1_AND_t2.bmp”파일을 2진 쓰기 모드로 열고, 파일을 만들 수 없으면 에러 메시지를 출력한다.
* 25,26: 읽기 파일에서 BMP의 헤더 54바이트를 읽는다.
* 27: 출력 파일에 BMP의 헤더 54바이트를 쓴다.
* 28\~32: 32바이트씩 2 개의 파일에서 읽어서 이 둘을 비트 AND 연산을 한 다음 출력 파일에 기록한다.
* 34\~36: 읽기, 쓰기 파일을 닫는다.
> [!NOTE]
> 입력 파일은 여러분이 윈도우즈 그림판에서 아무렇게나 그려서 두 개의 파일을 따로 만들면 된다. 단, 여기서는 이 두 개의 파일의 사이즈가 동일해야 제대로 처리가 된다.
